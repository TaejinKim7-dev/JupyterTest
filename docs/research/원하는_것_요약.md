# 원하는 것 요약

## 한 줄 요약

JupyterLab 안에서 LLM을 사용해 Android/Linux `ramdump`와 `vmlinux`를 대화형으로 분석하고, 반복적인 커널 디버깅 작업을 자동화하는 시스템을 원한다.

## 내가 원하는 핵심 목표

- 사람이 수작업으로 하던 메모리 덤프 분석을 AI가 보조하거나 자동화하기
- 레지스터, 콜스택, 커널 로그를 자연어로 해석해 장애 원인을 빠르게 찾기
- 분석에 필요한 Python 스크립트와 도구 호출을 LLM이 자동으로 생성하게 만들기
- 분석 노하우를 템플릿과 워크플로우로 정리해 팀 전체가 재사용할 수 있게 하기

## 꼭 필요한 기능

- 대화형 장애 원인 분석
- 분석 스크립트 자동 생성
- 메모리 비트 플립 등 이상 패턴 탐지
- Jupyter 노트북 기반의 실험/검증 환경
- 팀이 함께 쓸 수 있는 공통 분석 인터페이스

## 기술적으로 원하는 방향

- JupyterLab을 기본 UI로 사용
- Python 기반 `LLMAssistant`와 IPython 매직 커맨드로 빠르게 시작
- 이후 MCP 서버를 붙여 LLM이 분석 함수를 직접 호출할 수 있게 확장
- 분석 백엔드로 `PyKdump`, `GDB`, `pyELFtools`, `Volatility3` 같은 도구 활용
- 로컬 또는 사내 LLM API와 연동

## 필수 제약 조건

- `ramdump`와 `vmlinux`는 외부로 유출되면 안 됨
- 가능하면 온프레미스 또는 로컬 LLM만 사용
- LLM에는 원본 메모리 전체가 아니라 필요한 분석 결과 요약만 전달
- 오프라인 또는 에어갭 환경에서도 동작할 수 있어야 함

## 우선순위

1. 사내 또는 로컬 LLM 연결
2. `LLMAssistant` 헬퍼 클래스 구현
3. Jupyter 매직 커맨드(`%%ai`, `%%ai_analyze`) 추가
4. 프롬프트 템플릿 정리
5. MCP 기반 표준 분석 도구로 확장

## 기대 효과

- 수작업 분석 시간을 크게 줄임
- 숙련도가 낮은 팀원도 분석 흐름을 따라갈 수 있음
- 반복 작업을 자동화해 커널 디버깅 생산성을 높일 수 있음
- 분석 지식을 개인 의존이 아니라 팀 자산으로 바꿀 수 있음
