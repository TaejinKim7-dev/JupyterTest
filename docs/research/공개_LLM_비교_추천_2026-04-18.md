# 공개 LLM 비교 및 추천

기준일: 2026-04-18

대상 시나리오:

- Jupyter + memory dump 분석
- 코드 생성
- 구조화된 JSON 출력
- 향후 tool calling / agent 확장

공식 문서 기준으로 확인한 결과, 현재 프로젝트에 가장 현실적인 선택지는 `Groq`, `OpenRouter`, `Together AI` 3개다.

## 결론

| 순위 | 추천 | 이유 |
|---|---|---|
| 1 | `Groq + openai/gpt-oss-120b` | OpenCode 공식 연동, 매우 빠름, OpenAI-compatible, strict structured outputs 지원 |
| 2 | `OpenRouter + openai/gpt-oss-120b:free` 또는 `openrouter/free` | 무료 실험 가능, free 모델 풀과 라우팅이 강점 |
| 3 | `Together AI + DeepSeek-V3.1` | OpenAI-compatible, function calling/JSON mode 지원, 운영형 확장에 유리 |
| 4 | `Groq + openai/gpt-oss-20b` | 가장 빠르고 저렴하게 반복 실험하기 좋음 |
| 5 | `Together AI + Qwen3.5-9B` | 매우 저렴한 테스트용 대안 |

## 추천 순서

### 1. 가장 추천

`Groq + openai/gpt-oss-120b`

이유:

- Groq가 OpenCode용 추천 모델로 직접 명시
- OpenCode가 Groq의 모든 모델을 지원한다고 문서에 나옴
- OpenAI-compatible이라 현재 코드에 `base_url`과 `api_key`만 바꾸면 됨
- `gpt-oss-120b`가 strict structured outputs 지원
- 필요시 built-in code execution까지 있음

### 2. 돈 안 쓰고 먼저 보기

`OpenRouter + openai/gpt-oss-120b:free`

백업:

- `openrouter/free`

이유:

- 무료
- OpenAI SDK 그대로 연결 가능
- structured outputs/tool calling 요구사항에 맞는 free 모델을 라우터가 골라줌
- 대신 free는 모델 고정성과 응답 일관성이 떨어질 수 있음

### 3. 조금 더 운영형

`Together AI + DeepSeek-V3.1`

이유:

- OpenAI-compatible
- function calling, JSON mode 지원
- DeepSeek 페이지에서 no data retention by default와 privacy controls를 명시
- 비용도 아직 감당 가능한 편

단, 무료 시작은 아니고 현재 최소 `$5` 충전이 필요하다.

## 중요 메모

### 1. Qwen3.6 Plus free는 현재 고정 추천하기 어렵다

검색 캐시에는 `Qwen3.6 Plus (free)`가 보였지만, 현재 OpenRouter의 Qwen 페이지 기준으로는 `Qwen3.6 Plus`가 유료로 표시된다.

- `1M context`
- `$0.325/M input`
- `$1.95/M output`

즉, 최근에 free 상태가 바뀌었거나 종료된 것으로 보인다.

### 2. dump 분석 프로젝트에서는 구조화 출력 안정성이 더 중요하다

이 프로젝트는 단순 채팅이 아니라 다음이 필요하다.

- 분석 컨텍스트 요약
- JSON 결과
- tool calling
- 후속 스크립트 생성

그래서 cheapest 모델보다 `gpt-oss-120b` 계열이 더 잘 맞는다.

### 3. public LLM 사용 시 원본 dump 전체 전송은 피하는 것이 맞다

권장 원칙:

- 원본 dump 전체 전송 금지
- 요약, 발견 패턴, 콜스택, 문자열 일부만 전송
- 필요 시 provider policy를 강제
